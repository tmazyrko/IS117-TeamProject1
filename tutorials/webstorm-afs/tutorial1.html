    How to set up WebStorm to auto-save and deploy to AFS

    AFS (Andrew File System) is a fantastic tool that allows us to store files on servers for the purposes of backing them up, accessing them remotely, and even web hosting.

    Normally, AFS is accessed through terminal commands as it primarily runs on Linux, but we can set up WebStorm to automatically upload the latest versions of files in our projects to a specified user account and directory in AFS through a graphical interface. For the purposes of this tutorial, we will be using NJIT AFS.

    The AFS features and configuration in WebStorm can be found by navigating to the Tools menu, and then opening the Deployment sub-menu. To enable AFS deployment for your current project, open the Configuration window as shown below.

    In the deployment configuration window, click the ‘+’ icon to add a new remote connection. Set the type to ‘SFTP’ and fill out the hostname for your AFS server. Next, enter your AFS login information, and click ‘Test Connection.’ If the test is successful, proceed to filling out the root path box—this is the AFS directory where your project folder will be uploaded. In most cases, using the autodetect feature will fill in the root path, but you may have to add subdirectories if you do not want your project to reside in the root of your AFS account. ‘Web server URL’ is only required if your AFS server supports web hosting.

    The final step is to set the paths in the mapping tab. ‘Local path’ should point to your local project directory, while ‘Deployment path’ and ‘Web path’ are the relative paths of where your project will be on AFS.

    Optionally, you can have WebStorm automatically upload to AFS whenever a file is updated and saved. To enable this, navigate once more to the ‘Deployment’ menu and enable “Automatic Upload.”
