    Building an HTML5 Boilerplate website with Gulp

    The easiest way to build your HTML5 Boilerplate website with Gulp is to add a new Run/Debug Configuration in WebStorm. In the top right corner of the WebStorm window, click “Add Configuration…” Then, open the Templates dropdown, select Gulp.js, and in the top right corner of the Configuration windows click “Create configuration.” If the Gulpfile box is not already filled in, press the dropdown arrow and select the only option available. Then, in Tasks, write “build” (no quotation marks), and press Ok to close the window. Now, whenever you press the green play button in the top right corner, the contents of the src folder will be built and the results will be placed in the dist folder.

    Deploying your built website to AFS

    To deploy a built website, we only want to upload the contents of the dist folder to AFS. Follow our tutorial on how to set up WebStorm to auto-save and deploy to AFS, but when you are configuring the “Mappings” tab, add “/dist” to the end of the Local path. This will ensure than only the dist folder containing the built version of the website will be uploaded to AFS for hosting.
